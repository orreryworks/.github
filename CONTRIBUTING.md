# Contributing to Orreryworks

Thank you for your interest in contributing! This document provides general guidelines that apply across all **orreryworks** repositories.

Individual repositories may have their own `CONTRIBUTING.md` with repo-specific instructions for building, testing, and development setup. Always check the repository you're contributing to for additional guidance.

## General Guidelines

### Before You Start

- **Search existing issues** first to avoid duplicates.
- **Open an issue before starting non-trivial work** to discuss the approach and make sure it aligns with the project's direction.

### Submitting Changes

1. **Fork the repository** and create a branch from `main`.
2. **Keep changes focused** — one logical change per pull request.
3. **Write clear commit messages** that describe what and why.
4. **Add or update tests** for any new or changed behavior.
5. **Update documentation** if your change affects user-facing behavior.
6. **Make sure CI passes** before requesting review.

### Pull Request Process

- Reference any related issues (e.g., `Closes #42`).
- Provide a clear description of what the PR does and why.
- Be responsive to review feedback.

### Reporting Bugs

- Use the **Bug Report** issue template if one is available.
- Include steps to reproduce the problem.
- Include the expected and actual behavior.
- Include version information where relevant.

### Suggesting Features

- Use the **Feature Request** issue template if one is available.
- Describe the use case and why the feature would be valuable.
- Include examples showing how it would look or work.

## Code of Conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
