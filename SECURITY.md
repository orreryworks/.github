# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Orrery, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please use [GitHub's private vulnerability reporting](https://github.com/orreryworks/orrery/security/advisories/new) to submit your report. This ensures the issue can be assessed and addressed before public disclosure.

### What to Include

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (if you have one)

### What to Expect

- We will acknowledge your report within a reasonable timeframe.
- We will work with you to understand and resolve the issue.
- Once a fix is available, we will coordinate disclosure with you.

## Supported Versions

As the project is in early development, security fixes will be applied to the latest version only.

| Version | Supported |
| ------- | --------- |
| latest  | ✅        |

## Scope

This policy applies to all repositories under the [orreryworks](https://github.com/orreryworks) organization.
